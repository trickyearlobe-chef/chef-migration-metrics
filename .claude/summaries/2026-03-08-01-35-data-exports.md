## Context

The Data Exports feature enables users to export node readiness, blocked node, and cookbook remediation data in CSV, JSON, and Chef search query formats. The spec lives in `.claude/specifications/todo/visualisation.md` § Data Exports. Existing infrastructure included the `export_jobs` DB table (migration 0001), `ExportsConfig` in config, placeholder routes in the router, and reusable `filterNodes()`/`resolveOrganisationFilter()` helpers.

## What Was Done

### 1. Datastore Layer — `internal/datastore/export_jobs.go`
- `ExportJob` struct matching all 14 columns in the `export_jobs` table.
- Status/type/format constants (`pending`, `processing`, `completed`, `failed`, `expired`; `ready_nodes`, `blocked_nodes`, `cookbook_remediation`; `csv`, `json`, `chef_search_query`).
- `InsertExportJob`, `GetExportJob`, `UpdateExportJobStatus`, `UpdateExportJobExpired`, `ListExportJobsByStatus`, `ListExpiredExportJobs`, `ListExportJobsByRequester`.
- `ValidExportType()`, `ValidExportFormat()` validation helpers.
- Row scanning helpers following the project's existing patterns.

### 2. Export Package — `internal/export/`
- **`export.go`** — `ExportResult` struct, `DataStore` interface (subset of `datastore.DB`).
- **`filter.go`** — Shared `Filters` struct with `FilterNodes`, `FilterOrganisations`, `FilterComplexities`, `ParseFilters`, `MarshalToJSON`. Extracted from `handle_nodes.go`'s `filterNodes()` so both API handlers and export generators share the same logic.
- **`ready_nodes.go`** — `GenerateReadyNodeExport` producing CSV, JSON, or Chef search query (`name:node1 OR name:node2`). Iterates orgs → nodes → readiness, filters, collects ready nodes, renders to chosen format. Supports both in-memory (sync) and file-on-disk (async) output.
- **`blocked_nodes.go`** — `GenerateBlockedNodeExport` producing CSV or JSON. Includes blocking cookbook names (parsed from JSONB), blocking reasons (derived from readiness booleans), and aggregate complexity scores (looked up via `ListCookbookComplexitiesForOrganisation`). CSV uses semicolons for array fields.
- **`cookbook_remediation.go`** — `GenerateCookbookRemediationExport` producing CSV or JSON. Joins cookbooks with complexity records, applies target version and complexity label filters.
- **`cleanup.go`** — `CleanupExpiredExports` finds expired completed jobs, deletes files, marks jobs expired. `StartCleanupTicker` runs cleanup on a configurable interval in a background goroutine.

### 3. Web API Handlers — `internal/webapi/handle_exports.go`
- `POST /api/v1/exports` — validates request body (export_type, format, target_chef_version), estimates row count, dispatches to sync (HTTP 200 with file stream) or async (HTTP 202 with job ID) path based on `AsyncThreshold`.
- `GET /api/v1/exports/:id` — returns job status JSON with download_url when completed.
- `GET /api/v1/exports/:id/download` — serves the file with Content-Disposition; returns 409 if not completed, 410 if expired.
- Async export runs in a goroutine, updates job status, broadcasts WebSocket events (`export_started`, `export_complete`, `export_failed`).

### 4. Router + Interface Updates
- `internal/webapi/router.go` — replaced placeholder `handleNotImplemented` with `handleExports` and `handleExportStatus`.
- `internal/webapi/store.go` — added 6 export job methods to `DataStore` interface.
- `internal/webapi/store_mock_test.go` — added 6 corresponding mock fields and method implementations.

### 5. Frontend
- **`frontend/src/types.ts`** — `ExportType`, `ExportFormat`, `ExportJobStatus`, `ExportFilters`, `ExportRequest`, `ExportJobResponse`.
- **`frontend/src/api.ts`** — `createExport()` (handles 200 sync download + 202 async), `fetchExportStatus()`, `downloadExportUrl()`.
- **`frontend/src/components/ExportButton.tsx`** — Self-contained button with format dropdown picker, sync/async flow handling, polling with timeout, download trigger, error display.

### 6. Tests
- **`internal/datastore/export_jobs_test.go`** — 16 tests covering constants, JSON marshalling, validation helpers, zero values, lifecycle.
- **`internal/export/export_test.go`** — 25+ tests with a `fakeStore` covering all three generators (CSV/JSON/search query), all filters (environment, platform, role, organisation, complexity label, target version), MaxRows, write-to-disk, empty stores, blocking cookbook parsing, blocking reason derivation, cleanup (file delete + mark expired, missing file, no expired jobs), and filter roundtrip.
